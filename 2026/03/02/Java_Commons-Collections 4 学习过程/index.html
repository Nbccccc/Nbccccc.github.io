这里我们就要先了解下知识了

PriorityQueue：

PriorityQueue是一个优先队列，即每次出队的元素都是优先级最高的元素。jdk中使用二叉堆这种数据结构：

通过堆使得每次出队的元素总是队列里面最小的，而元素的大小比较方法可以由用户Comparator指定，这里就相当于指定优先级。

PriorityQueue在反序列化时会调用heapify() 方法来恢复堆结构的顺序，heapify() 中调用siftDown() ，在siftDown() 中当用户指定了Comparator，就会调用siftDownUsingComparator()进行排序，它是将⼤的元素下移。

siftDownUsingComparator() 中使⽤Comparator().compare() ⽅法⽐较树的两个节点，此时传入的Comparator为TransformingComparator，由此触发Transformer#transform()造成命令执行。

关于PriorityQueue的详细实现原理，可以参考这篇文章：

https://www.cnblogs.com/linghu-java/p/9467805.html

by Chat GPT

这段代码是在实现堆（Heap）数据结构中的一个方法，该方法用于将一个无序的数组调整为堆的形式。通常堆是通过数组来实现的，数组的索引从0开始，而堆的性质是：对于任意节点i，其左子节点索引为2i+1，右子节点索引为2i+2，父节点索引为(i-1)/2。

在堆化过程中，我们从数组的倒数第二层的最后一个节点开始向上遍历，对每一个节点执行一次下沉（siftDown）操作，以保证节点满足堆的性质。倒数第二层的最后一个节点的索引是(size >>> 1) - 1，这是因为：

  • size >>> 1 是将 size 向右移动一位，相当于将 size 除以 2，表示的是最后一层的节点数量，也就是叶子节点的数量。
  • 减去 1 是因为堆的索引是从0开始的，所以需要将叶子节点的数量减1得到倒数第二层的最后一个节点的索引。

因此，(size >>> 1) - 1 计算出的是倒数第二层的最后一个节点的索引，从这个节点开始往前遍历，对每个节点执行下沉操作，完成堆的构建。

所以Size 就是 PriorityQueue 这个队列的长度，现在我们这个队列的长度为 0，0 - 1 = -1，所以会直接跳出循环，不能弹计算器。
